The node, cluster, ingress, container registry, DNS, image build, and Deployment spec are all Terraform, applied through HCP Terraform in a private repo. The site itself is a static HTML/CSS page.

Push to merge, live in ~44 seconds — build, registry push, and a rollout pinned to the commit SHA.

AI wrote the HTML/CSS on this page. I do infra, not frontend.

platform
01

cluster

One Talos node. Control plane and worker, same physical server (allowSchedulingOnControlPlanes). No managed Kubernetes underneath. The node is provisioned out-of-band; Terraform bootstraps it with the siderolabs/talos provider and stores the resulting kubeconfig as state.

02

registry

Self-hosted Harbor on the same cluster. This site gets its own harbor_project. Pushes and pulls go through a CI robot account whose password is a Terraform random_password — never typed by hand — packaged into a dockerconfigjson secret and mounted as an imagePullSecret.

03

ingress & tls

ingress-nginx runs as a DaemonSet with hostNetwork: true, bound to the node's public IP. No load balancer. Each domain has its own cert-manager ClusterIssuer solving via DNS-01 against Cloudflare. Zone SSL mode is strict, so TLS is independently valid on both hops: browser→edge and edge→origin.

the site
04

image

Two-stage Dockerfile. Stage one is Alpine + imagemagick, converting the source PNG into a multi-resolution favicon.ico (16/32/48/64). Stage two is a pinned nginx:1.29-alpine with nginx.conf, the static HTML, and the favicon copied in. The single server block layers a security-header set (HSTS, CSP, nosniff, frame-deny, referrer policy), a /healthz endpoint, and gzip on top of a try_files fallback that redirects unknown paths back to /.

05

deployment

Two replicas behind a ClusterIP Service. 50m CPU / 16Mi requested, 200m / 64Mi limit. Startup, readiness, and liveness probes all hit /healthz. Rollouts surge to 100% with zero unavailable plus a pre-stop drain, and a PodDisruptionBudget keeps at least one replica serving. A VerticalPodAutoscaler in Auto mode tunes the memory request in that band; CPU is left unmanaged.

06

build & deploy

A push to the site's path on prd (or a manual workflow_dispatch) triggers GitHub Actions: log into Harbor, buildx build, and push two tags — :latest and the commit SHA. The rollout is a kubectl set-image to the immutable SHA tag followed by rollout status, so every release is a distinct pinned image and a rollback points somewhere real.

07

mirror

A second job rsyncs two things to a public GitHub repo via a scoped token: the site source into site-image/ and the workflow file into github-workflow/. It commits only if something changed. Source lives in two places — the private infra monorepo where it's built, and a public read-only mirror.

08

dns & cdn

vin.moe's A record points at the node's IP through a proxied Cloudflare record. Combined with DNS-01, none of Let's Encrypt's validation traffic reaches the origin — the only thing that must be reachable is the origin's own valid cert, and only Cloudflare needs to see it. The proxy also acts as the CDN: a Terraform-managed cache ruleset tells Cloudflare's edge to cache everything on this hostname — HTML included, not just static assets — so most requests never touch the origin at all.

node & security
09

the node

Dedicated hardware running Talos Linux — an immutable, API-driven OS with no shell, no SSH, and no package manager; every change goes through its config API, not a login. The ephemeral and state partitions are both LUKS2-encrypted. A host-level ingress firewall defaults to deny, with public web traffic accepted only from Cloudflare's published edge ranges — so the origin quietly ignores direct scans and floods.

10

defense in depth

Public traffic terminates at Cloudflare and is re-encrypted to the origin — strict SSL, independently valid on both hops — with HSTS and a content-security policy on every response. Administrative surfaces sit behind Authelia two-factor; the SSH VPS (debian pod deployed on cluster for remote dev) is reachable only over a private WireGuard mesh, never the public internet. In-cluster secrets are generated by Terraform and held in HCP Terraform, never committed to the repo, and CI runs on a namespace-scoped token rather than cluster-admin. Workloads run least-privilege under Pod Security Standards.

11

watch & recover

A Prometheus / Grafana / Loki / Tempo stack watches the node and every workload. Persistent volumes back up nightly to encrypted object storage on rolling retention, and dependency bumps land automatically through Renovate with a human gate on majors.
